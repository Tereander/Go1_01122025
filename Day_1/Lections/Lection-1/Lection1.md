# Лекция 1. Установка и запуск первой программы

## Шаг 1. Установка компилятора
* Переход по ссылке: https://go.dev
* Установили компилятор для своей ОС  
**Важно:** на данном курсе желательно, чтобы у вас была версия не ниже 1.18

## Шаг 2. GOROOT, GOPATH
**Определение**: ```GOROOT``` - это файловый пусть, указывающий на расположение **КОМПИЛЯТОРА** Go.  
**Определение**: ```GOPATH``` - это файловый пусть, указывающий на расположение **РАБОЧЕГО ОКРУЖЕНИЯ** (место, где мы пишем код и создаем проекты). По умолчанию, на курсе мы создали `GOPATH` по адресу: `/home/<username>/go`.


## Шаг 3. Инициализация рабочего окружения
Чтобы создать рабочее окружение, нам нужно в `GOPATH` определить 3 директории:  
1. `src` - место, где будут лежать исходники
2. `bin` - место, где будут лежать скомпилированные бинарники, после выполнения компиляции проектов
3. `pkg` - место, где будут жить сторонние пакеты для наших проектов

## Шаг 4. Первая программа
В `GOPATH/src/<git_site>/<repo_name>` создадим файл `main.go` со следующим содержимым: 
```
package main

import "fmt"


func main() {
	fmt.Println("Hello, world!")
}
```

## Шаг 5. Запуск и компиляция
Go - компилируемый язык.  
Для того, чтобы скомпилировать исполняемый файл, можно использовать следующую команду: 
```
go build <path/to/go/file.go>
```
Данная команда создаем исполняемый файл по месту её вызова. Это удобно, когда мы в пылу создания хорошей программы и хочется 
посмотреть, скомпилируется ли оно вообще или в случае каких-либо тестов - позволяет на месте всё проверить.  

Другая команда, которая также позволяет создать исполняемый файл:
```
go install <path/to/go/file.go>
```
Данная команда создаст исполняемый файл по пути `GOPATH/bin`.

Третья команда, которой мы будем часто пользоваться на курсе - `go run`:
```
go run <path/to/go/file.go>
```
Данная команда делает следующие действия:
* Создаёт исполняемый файл во временном хранилище
* Запускает этот файл
* И "устраняет" его (завист от ОС)

Для того, чтобы узнать, где это временное хранилище: `--work`

## Шаг 6. Правильная структуризация рабочего окружения
Рекомендую создать следующий путь:
```
GOPATH/src/git_server/<your_account/<your_repo_name>
```
